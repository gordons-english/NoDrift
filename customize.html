User communication profile

Customize how NoDrift works with you

Configure how Codex communicates, leads work, asks questions, explains decisions, and reports evidence without changing what Codex must verify.

Token savings should come only from communication-style choices, never from reducing verification, source fidelity, privacy protection, approval gates, or external-action safeguards. NoDrift does not attach its workspace files to outgoing messages.

How this works in Codex

Use the same profiles after installation

  1. Add the NoDrift files to your Codex project.
  2. Start the first Codex chat with begin or say Customize NoDrift.
  3. Codex offers Fast Work, Standard Work, Guided Work, and Deep Support; Standard Work is the balanced default.
  4. The chosen profile is active immediately. Saving it for future chats is optional and can happen later.
  5. By default, Codex leads with approval: it gives one clear next directive and asks for go before permission-sensitive steps.
  6. If your new-chat settings have not already handled it, Codex asks once whether you want a private Living Topic Map for the conversation.
  7. Optional verification comes later; Codex should state what it will check and how long it expects to take before starting.
  8. Use /goal later when you want Codex to keep driving toward a defined result. Goal mode increases persistence, not permission.

Only one profile is active at a time. Codex should ask before saving, changing, renaming, or deleting a durable profile.

Customization changes communication style and workflow support. It does not weaken NoDrift governance or add the full workspace package to each turn.

Workflow settings

How NoDrift works with you

These settings control how Codex moves through a project: who suggests the next task, how often Codex stops to ask questions, and when Codex needs your permission before changing files, publishing, deleting, or taking outside action.

Token impact: light. These settings mostly control workflow and approval behavior.
