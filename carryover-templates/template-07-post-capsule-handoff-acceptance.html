Template 07

Post-Capsule Handoff Acceptance

Use this as the transfer test. The new chat must answer before it assumes responsibility, and the original chat can review the answers.

User Instructions

Paste this after the capsule or keep it as the final section of the capsule. The new chat must answer the questions before project work continues. The user can then paste the answers back into the original chat for review.

This creates a two-chat verification loop. The new chat proves what it understands. The original chat reviews whether the transfer is safe, incomplete, or confused. This does not authorize file work by itself.

Use this when

  • A new chat is expected to take over a project or branch.
  • The original chat remains the archive and reviewer.
  • The handoff contains important boundaries, approved wording, open questions, or risks.

Do not use this when

  • The new chat has not received the capsule.
  • The user does not want a handoff review loop.
  • The new chat is not expected to continue the project.

Original Chat Review Instruction

INSTRUCTION TO ORIGINAL CHAT REVIEWER

Review the new chat's handoff answers against the original capsule and available verified context.

Classify the transfer as:
- ACCEPTED: New chat can proceed within stated limits.
- ACCEPTED WITH CORRECTIONS: New chat can proceed after corrections are pasted into it.
- NOT READY: New chat missed important context, boundary, or source status.
- BLOCKED: Critical context is missing, conflicting, or unsafe.

Reviewer checks:
1. Did the new chat identify the controlling governance pointers?
2. Did it avoid treating the capsule as file truth?
3. Did it identify actions not authorized?
4. Did it preserve exact approved wording and decisions?
5. Did it label unknowns and unverified items?
6. Did it exclude unrelated project context?
7. Did it understand archive lookup limits?
8. Did it identify current files and artifacts?
9. Did it identify open questions and risks?
10. Did it state a safe next action?
11. If compaction occurred, did it separate file-verified facts from compacted-summary claims?
