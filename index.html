How No Drift Operates

What is No Drift?

No Drift is a downloadable workspace control kit for Codex. It gives a project operating rules, source checks, memory files, approval boundaries, and proof records.

Is this software or a separate app?

No. It is not a dashboard or background app. It operates through structured files, templates, and setup prompts that Codex reads inside the project workspace.

Is this just a prompt pack?

No. Prompts are included, but No Drift is a workspace governance system: root instructions, governance modules, approval gates, memory templates, checklists, and proof records.

Does it control the AI model?

No. No Drift does not control the LLM itself. It controls what the workspace accepts, labels, verifies, records, and acts on.

How do I use it after download?

Add the files to your Codex workspace, run the setup prompt, and ask Codex to verify the installed rules, memory files, source register, and safest next step.

What happens in a normal session?

You give Codex the work. Codex checks the project rules, keeps source boundaries visible, records important decisions, flags uncertainty, and pauses before unclear external action.

Do I fill out the files by hand?

Not as paperwork. The files are working templates for Codex-assisted projects. Codex can help maintain source registers, checkpoints, decisions, risks, and work queues as the project develops.

How long does setup take?

The basic setup is designed to take about five minutes. Deeper customization can happen later when the project has real sources, risks, decisions, and operating preferences.

Can non-Codex users use it?

The V1 English edition is Codex-specific. A platform-neutral edition is planned later; other AI tools may need adaptation before the files and rules fit cleanly.

Customization And Memory

Can I customize how Codex talks to me?

Yes. The user profile can set communication style, detail level, progress updates, teaching mode, uncertainty reporting, question style, and token-use preferences.

Does customization weaken the rules?

No. Communication preferences change how Codex talks. They do not override source fidelity, verification, privacy, approval gates, or the rule against inventing facts.

Can I customize memory templates?

Yes. A coding project may need implementation logs and test status. A writing or research project may need source notes, continuity records, and decision history.

What are project memory files?

They hold the current state, source register, source authority map, open decisions, work queue, checkpoints, risks, handoffs, and action logs outside fragile chat history.

What are carryover capsules?

Carryover capsules help a new chat continue responsibly. They identify what to read, approved decisions, open questions, risks, boundaries, next actions, and what not to change.

Can carryover capsules be customized?

Yes. The carryover template set can be adapted for project handoffs, branch discussions, archive lookup, omission checks, source coverage, and post-handoff readiness review.

Proof, Limits, Payment, And Delivery

Does No Drift guarantee no hallucinations?

No. It gives Codex rules and habits that make unsupported claims easier to catch before they become project facts. It reduces risk; it does not promise perfection.

Metrics apply to defined work periods. They can track confirmed drift, confirmed hallucinations, unauthorized actions, project contamination, communication quality, and productive output.
