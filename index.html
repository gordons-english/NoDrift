NoDrift Does Not Control AI.

In human communication, we do not control the transmission. We control the reception: what we accept, verify, record, and act on.

NoDrift applies that same discipline to AI-assisted project work.

  • It does not change the LLM or guarantee every answer is true.
  • It checks important claims before they become project facts.
  • It keeps decisions, open questions, and source boundaries visible.
  • It pauses unclear actions until approval is clear.
  • It turns recognized errors into private corrective lessons for future work.

How NoDrift Operates

What is NoDrift?

NoDrift is a downloadable workspace control kit for Codex. It gives a project operating rules, source checks, memory files, approval boundaries, and evidence records.

Is this software or a separate app?

No. It is not a dashboard or background app. It operates through structured files, templates, and setup prompts that Codex reads inside the project workspace.

Is this just a prompt pack?

No. Prompts are included, but NoDrift is a workspace governance system: root instructions, governance modules, approval gates, memory templates, Living Topic Maps, checklists, and evidence records.

Does it control the AI model?

No. NoDrift does not control the LLM itself. It controls what the workspace accepts, labels, verifies, records, and acts on.

How do I use it after download?

Add the files to your Codex workspace, run the setup prompt, and ask Codex to verify the installed rules, memory files, source register, and safest next step.

What happens in a normal session?

You give Codex the work. Codex checks the project rules, keeps source boundaries visible, records important decisions, updates an authorized Living Topic Map, flags uncertainty, and pauses before unclear external action.

Do I fill out the files by hand?

Not as paperwork. The files are working templates for Codex-assisted projects. Codex can help maintain source registers, checkpoints, decisions, risks, work queues, and private topic maps as the project develops.

How long does setup take?

The basic setup is designed to take about five minutes. Deeper customization can happen later when the project has real sources, risks, decisions, and operating preferences.

Can non-Codex users use it?

The V1 English edition is Codex-specific. A platform-neutral edition is planned later; other AI tools may need adaptation before the files and rules fit cleanly.

Personal Training And Defense Shield

What is the personal NoDrift training and defense shield?

It is the combined NoDrift system of governance, sources, project memory, checkpoints, incident records, corrective lessons, and reports. It makes recognized errors visible, correctable, and available as lessons for future work.

How much of the training system do I maintain?

Very little during regular work. You question anything that does not sound right, correct the review when needed, make important decisions, and answer whether a reported incident is Critical or Not Critical. The NoDrift review process maintains the routine records and reports.

What should I do when something does not sound right?

Question it critically and politely. The goal is not to prove anyone wrong; it is to identify and correct the error. Optional user request: “That does not sound right. Review the response under NoDrift and report whether an error occurred.”

What happens when NoDrift review recognizes an error?

It reports the relevant user message and workspace response or action, identifies the supported error type, scores the impact, prepares the correct response, links the applicable governance rule, and identifies affected work.

What do Critical and Not Critical mean?

Critical stops the affected work and requires your direction. Not Critical means the incident is recorded and corrected while already authorized work may continue.

How does this guard against lost information?

The complete incident record preserves the exact problem, source truth, correction, governing rules, affected work, and recurrence history. Active lessons, checkpoints, topic maps, and reports give later chats a required path back to that information.

What reports do I receive?

NoDrift defines reception-side reports for immediate alerts, task-completion reports, project checkpoints, periodic operations reports, and on-demand reports focused on decisions, risks, corrections, and next actions.

Does the shield guarantee perfect AI behavior?

No. It reduces risk, preserves corrections, and makes repeated known failures easier to identify. It does not guarantee that every mistake will be prevented or detected.

Customization And Memory

Can I customize how Codex talks to me?

Yes. The user profile can set communication style, detail level, progress updates, teaching mode, uncertainty reporting, question style, and token-use preferences.

Does customization weaken the rules?

No. Communication preferences change how Codex talks. They do not override source fidelity, verification, privacy, approval gates, or the rule against inventing facts.

Can I customize memory templates?

Yes. A coding project may need implementation logs and test status. A writing or research project may need source notes, continuity records, and decision history.

What are project memory files?

They hold the current state, source register, source authority map, open decisions, work queue, checkpoints, risks, handoffs, and action logs outside fragile chat history.

What are carryover capsules?

Carryover capsules help a new chat continue responsibly. They identify what to read, approved decisions, open questions, risks, boundaries, next actions, and what not to change.

Can carryover capsules be customized?

Yes. The carryover template set can be adapted for project handoffs, branch discussions, archive lookup, omission checks, source coverage, and post-handoff readiness review.

What is a Living Topic Map?

It is a private navigation file for an active conversation. It records important topics, approved decisions, exact wording, open questions, boundaries, files, risks, next actions, and branch relationships.

How do automatic topic branches work?

When authorized, Codex identifies a substantive tangent, creates a hierarchically named branch map, and registers it in the parent without treating it as a Git branch.

Does a Living Topic Map save every word?

No. It protects the important operating information needed to navigate and continue the work. The original chat remains the source for conversation detail that was never recorded.
