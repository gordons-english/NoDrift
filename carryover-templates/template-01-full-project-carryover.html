Template 01

Full Project Carryover Capsule

Use this when an active project must move into a new working chat without losing project state, approved decisions, source boundaries, file status, risks, or next actions.

User Instructions

Fill this template with project-specific source material. Do not paste the full operating governance unless the user explicitly asks. Instead, point to the governance files or protocol the new chat must read before acting.

This template is for a complete continuity handoff, not a short starter prompt. Include full known operative detail. If important information is unavailable, label it UNKNOWN, NOT VISIBLE, or NOT INSPECTED. If the capsule becomes long, split the filled capsule into numbered parts instead of compressing important content.

Use this when

  • A project has become too large or slow in the original chat.
  • A new project-specific chat must continue file-aware work.
  • Approved wording, status, QA, boundaries, or decisions may be lost without a full handoff.

Do not use this when

  • The new chat is only for casual discussion. Use the Branch / Discussion template instead.
  • The project facts are not visible and cannot be verified. Use UNKNOWN labels or block.
  • The user has asked for analysis only and has not authorized creating a handoff.

End Directive And Readiness Questions

DIRECTIVE TO THE NEW CHAT

Before assuming responsibility for this project, answer the questions below. The user will take your answers back to the original chat for review. Do not edit, create, move, delete, stage, commit, publish, upload, send, share, deploy, or configure anything until the handoff is accepted or the user gives explicit task approval.

Governance understanding questions:
1. Which governance files or protocols control your operation?
2. Does this capsule replace those governance files?
3. What actions are not authorized by this capsule?
4. What must you verify before editing files or claiming current status?
5. What source labels will you use for uncertain or unverified information?
6. What do you do if current files conflict with this capsule?
7. What requires explicit user approval?
8. When may you consult the original archive?
9. What unrelated project context must you exclude?
10. What would make you stop instead of proceed?

Source-material understanding questions:
1. What project are you continuing?
2. What is the workspace path?
3. What is the physical project folder?
4. What is the project purpose?
5. What is the current status?
6. What files or artifacts are involved?
7. What exact approved wording or decisions must be preserved?
8. What work has already been completed?
9. What verification has already been performed?
10. What remains unverified?
11. What open questions remain?
12. What risks or unresolved confusion remain?
13. What is the next safe action?
14. What must not be done yet?
15. What information is missing before you can take responsibility?
