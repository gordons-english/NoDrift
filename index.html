NoDrift Does Not Control AI

In human communication, we do not control the transmission. We control the reception: what we accept, verify, record, and act on.

NoDrift applies that same discipline to AI-assisted project work.

  • It does not change the LLM or guarantee every answer is true.
  • It does not add NoDrift files to outgoing prompts or conversation turns.
  • It checks important claims before they become project facts.
  • It keeps decisions, open questions, and source boundaries visible.
  • It pauses unclear actions until approval is clear.
  • It turns recognized errors into private corrective lessons for future work.

NoDrift FAQ

Short answers for setup, operation, memory, customization, evidence, payment, and platform support.

Start Here

What is NoDrift?

NoDrift is a ready-to-copy governance workspace for Codex. It gives serious project work rules, source checks, memory records, approval boundaries, corrective lessons, and evidence records.

Is this software or a separate app?

No. NoDrift is not a dashboard, plugin, executable, or background monitor. It works through structured workspace files and setup prompts used inside a Codex project workspace.

Is this just a prompt pack?

No. Prompts are included, but the product is the workspace structure: governance files, approval matrix, source register, memory templates, Living Topic Maps, incident logs, checklists, and examples.

How do I start?

Download the workspace starter, unzip it, copy the included workspace files into your Codex workspace folder, open Codex there, and send begin. Codex should offer the quick working profiles first, then ask about a private topic map if needed, and run optional verification only after stating the scope and expected time.

Do I fill out the files by hand?

NoDrift is not meant to become paperwork. Codex helps maintain the records during work; the user answers questions, approves important actions, and questions anything that does not sound right.

How long does setup take?

The first setup pass is designed to be quick: copy the files, send begin, choose a working profile, and decide whether you want the optional topic map. Deeper customization and verification can happen later.

How It Works

Does NoDrift control the LLM?

No. The LLM generates the response. NoDrift governs reception: what the work session accepts, verifies, records, corrects, and acts on after output arrives.

Does it attach files to outgoing messages?

No. NoDrift does not attach governance files, memory files, corrective logs, or workspace files to outgoing messages or conversation turns.

What happens in a normal session?

The user explains the work. Codex works inside the project workspace. NoDrift keeps rules, sources, decisions, checkpoints, risks, branches, and approval boundaries visible.

What if something sounds wrong?

Question it directly and politely. NoDrift can treat the issue as a candidate error, preserve the relevant exchange, classify the likely problem, and prepare a correction path.

What does Critical mean?

Critical damage means an error may affect files, publishing, payments, external actions, or serious project direction. Important but non-damaging errors can still become corrective lessons.

Does NoDrift guarantee perfect answers?

No. NoDrift reduces risk, preserves corrections, and makes unsupported claims easier to catch. It does not guarantee that every mistake will be prevented or detected.

Customization, Memory, And Updates

Can I customize how Codex talks to me?

Yes. The working profile can set communication style, detail level, progress updates, teaching mode, uncertainty reporting, question style, token-use preferences, and whether Codex leads by recommending the next safe directive.

Does customization weaken the rules?

No. Customization changes communication and workflow style. It does not override source fidelity, privacy, approval gates, verification, coverage-table requirements for broad claims, or the rule against inventing facts.

What affects token use?

Communication detail has the largest effect. NoDrift files are not attached to outgoing messages; token use changes mainly when the user asks for more explanation, status detail, teaching, or evidence labels.

What are project memory files?

They preserve current state, source boundaries, open decisions, work queues, checkpoints, risks, handoffs, incident records, and action logs outside fragile chat history.

What is a Living Topic Map?

A Living Topic Map is a private navigation record for an active conversation. It preserves decisions, open questions, branches, files, risks, next actions, and status.

How do corrective lessons updates work?

Reusable correction patterns can be delivered as user-safe update files. The user adds the update to private project memory without replacing the full governance package.

What if something goes wrong?

If you are already working in your NoDrift workspace, type troubleshooting in Codex. The Codex agent should pause forward work, check only what is needed, explain what it found, and ask for go before changing anything.
