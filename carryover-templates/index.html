Carryover Capsule Templates

Use these templates to move project context between chats without rewriting governance, inventing missing facts, or losing important boundaries.

User Instructions

These pages are blank reusable templates. They do not contain project source material, and they do not replace your governing protocol. Before filling any template, identify the active workspace, the project folder, the governing files or protocol that the new chat must read, and the exact source material that is allowed for the handoff.

Fill the template with supported project information only. If something is missing, uncertain, unavailable, stale, or not inspected, label it directly. Do not use the template to compress important approved wording, current status, source rules, risks, open questions, or actions not authorized.

Recommended order

  1. Use the Source Coverage Report before or while creating any serious capsule.
  2. Use the Full Project or Branch template to carry the active context.
  3. Add Archive Lookup instructions only when old chat lookup may be needed.
  4. Add the Omission Report to prevent quiet compression.
  5. Paste the New Chat Starter Prompt after the capsule.
  6. Use the Post-Capsule Handoff Acceptance template before the new chat assumes responsibility.

Required source labels

  • FILE-VERIFIED
  • GIT-VERIFIED
  • TOOL-VERIFIED
  • CHAT-VISIBLE
  • USER-APPROVED
  • USER-REPORTED
  • PROPOSED
  • DISCUSSED
  • REJECTED
  • SUPERSEDED
  • UNKNOWN
  • NOT INSPECTED
  • NOT VISIBLE
