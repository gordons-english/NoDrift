Template 02

Branch / Discussion Chat Capsule

Use this when a side chat should handle discussion, strategy, wording, design, or exploration without becoming permission to change project files.

User Instructions

Fill this template when a new chat is meant to discuss a narrow branch of a project. The branch may think, compare, draft, or recommend, but it must not become a build or file-editing chat unless the user later authorizes that transition.

Be precise about what belongs in the branch and what must stay in the parent project chat. Any decision made in the branch that should affect project files must be copied back to the project working chat before file work begins.

Use this when

  • The user wants a lighter chat for discussion or strategy.
  • The parent project chat is too heavy for brainstorming.
  • You need to separate ideas from implementation permissions.

Do not use this when

  • The new chat is expected to assume full project responsibility. Use the Full Project template.
  • The branch will publish, send, deploy, or edit files immediately.
  • The branch boundary is unclear and could mix unrelated projects.

End Directive And Readiness Questions

DIRECTIVE TO THE NEW BRANCH CHAT

Before continuing, answer these questions. The user may take your answers back to the parent chat for review. Do not edit, create, move, delete, publish, send, share, stage, commit, deploy, or configure anything unless the user explicitly authorizes that specific action.

Governance understanding questions:
1. Is this branch allowed to edit files?
2. What must happen before branch discussion becomes project action?
3. What topics are excluded from this branch?
4. How will you label brainstormed ideas?
5. What requires return to the parent project chat?
6. What source material may you use?
7. What source material must you not use?
8. What do you do if this branch conflicts with the parent project files?
9. What external actions are not authorized?
10. What would make this branch unsafe to continue?

Source-material understanding questions:
1. What parent project does this branch belong to?
2. What is the branch purpose?
3. What topics are included?
4. What topics are excluded?
5. What approved decisions transferred into this branch?
6. What remains only proposed or discussed?
7. What files or artifacts, if any, may be referenced?
8. What must be copied back to the parent chat before implementation?
9. What is the next safe discussion action?
10. What information is missing or unknown?
