Template 04

Capsule Source Coverage Report

Use this to show exactly what sources were read, listed, searched, unavailable, or not inspected before trusting a carryover capsule.

User Instructions

Fill this report before or alongside any serious carryover capsule. It protects the new chat from assuming the capsule is more complete than it really is. A filename, folder name, or search result is not proof of content.

The report should be honest even when incomplete. It is better to say a source was listed but not inspected than to imply it was read. If critical sources are unavailable, the capsule may need to be labeled partial or blocked.

Use this when

  • A capsule depends on workspace files, git state, browser QA, old chat history, or tool output.
  • The user needs to know whether a handoff is safe to trust.
  • You need to separate verified state from chat-visible or unknown state.

Do not use this when

  • You have not inspected anything and cannot label coverage accurately.
  • The user asked only for a quick conversational answer, not a reusable handoff.

End Directive And Readiness Questions

DIRECTIVE TO THE NEW CHAT

Before relying on the capsule, answer these questions from the source coverage report. Do not treat uninspected sources as verified.

Governance understanding questions:
1. Which governance files were read?
2. Which governance files were only pointed to or not inspected?
3. Which claims are file-verified?
4. Which claims are chat-visible only?
5. Which sources were not inspected?
6. Are any claims based only on filenames or folder names?
7. What do you do with conflicts?
8. What must be verified before file work?
9. What does the coverage conclusion allow you to do?
10. What does the coverage conclusion prohibit?

Source-material understanding questions:
1. What core project files were read?
2. What artifacts were not checked?
3. What current status is verified?
4. What current status is unverified?
5. What source material is unavailable?
6. What source material was intentionally not inspected?
7. What conflicts were found?
8. What limitations affect the capsule?
9. What must be checked before the next action?
10. Is the capsule safe, partial, blocked, or not safe?

Final readiness statement:
Choose one:
- COVERAGE ACCEPTED WITH LIMITATIONS: [state limits]
- NEEDS MORE VERIFICATION: [state sources]
- BLOCKED: [state missing source]
